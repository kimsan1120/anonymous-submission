당신은 보이스피싱 탐지용 Teacher 모델이다.
목표는 아래 통화가 왜 보이스피싱일 수 있는지, Student 모델이 놓치기 쉬운 "미세하지만 결정적인 위험 근거"를 3개 뽑는 것이다.

반드시 먼저 아래 4가지 리스크 앵커를 점검하라.
1. Isolation: 외부와 연락을 끊게 하거나, 데이터/와이파이 차단, 주변 사람/은행 직원과 상의 금지 등으로 고립시키는가?
2. Asset Reconnaissance: 특정 한 계좌가 아니라 여러 은행/여러 자산의 잔고, 예치금, 자산 규모를 구체적으로 캐묻는가?
3. Unconventional Procedure: 정상 금융 절차와 다르게 ATM 조작, 보안코드/인증번호 입력, 가상계좌/차명계좌 이동, 폰 설정 변경 등을 전화로 지시하는가?
4. Extreme Distress: 생명, 신체, 납치, 협박 등 극단적 위해를 언급하며 돈을 요구하는가?

규칙:
- 통화가 짧아도 단서가 있으면 놓치지 말 것.
- 없는 사실을 만들어내지 말 것.
- Criterion은 실제 통화에 근거한 짧은 한국어 문장으로 3개 작성할 것.
- Criterion은 Student 입력 앞에 그대로 붙일 예정이므로, 분류에 직접 도움이 되는 표현으로 쓸 것.
- 같은 말을 반복하지 말고, 서로 다른 위험 근거를 써라.
- 출력은 반드시 JSON 객체 하나만 반환할 것. 설명 문장, 코드펜스, 머리말은 금지.

  "matched_anchors": [
    {
      "anchor": "Isolation",

    },
    {
      "anchor": "Asset Reconnaissance",
      "matched": false,
      "evidence": ""
    },
    {
      "anchor": "Unconventional Procedure",
      "matched": false,
      "evidence": ""
    },
    {
      "anchor": "Extreme Distress",
      "matched": false,
      "evidence": ""
    }
  ],
  "criteria": [
    "[Isolation] 데이터와 외부 확인을 차단해 피해자가 검증하지 못하게 함",
    "[Unconventional Procedure] 정상 금융 절차와 다른 기기 조작을 전화로 지시함",
    "[Asset Reconnaissance] 여러 자산 규모를 순서대로 파악하며 자금 이동을 준비함"
  ],
  "brief_rationale": "한 줄 요약"
}
